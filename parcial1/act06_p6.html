Implementación IPSec VPN

Configuración de túnel VPN con IPSec en routers Cisco

2. Introducción

Actualmente, muchas organizaciones requieren conectar redes remotas de forma segura a través de Internet, evitando que la información sea interceptada o modificada. En esta actividad se implementó una VPN IPSec tipo Tunnel, configurando routers Cisco para establecer un canal cifrado entre dos redes privadas. A través de distintas fases, se aplicaron políticas ISAKMP, transform-set, ACLs y crypto maps, logrando una comunicación protegida que garantiza confidencialidad, integridad y autenticación en el intercambio de datos.

3. Configuración de la topología de red

En esta actividad se implementó una VPN IPSec tipo Tunnel para asegurar la comunicación entre dos redes privadas, conectadas a través de un ISP. Se siguieron diferentes fases de configuración para asegurar conectividad, habilitar criptografía y establecer el túnel.

Figura 1. Topología de red utilizada para la VPN IPSec.

4. Fases de configuración

4.1 Configuración inicial

Primero se prepararon los routers configurando hostname, interfaces, direcciones IP y rutas estáticas. Se activaron las interfaces con no shutdown y se agregó una ruta por defecto para permitir salida hacia el ISP. 

enable
configure terminal
hostname R1
interface g0/1
 ip address 192.168.1.1 255.255.255.0
 no shutdown

interface g0/0
 ip address 209.165.100.1 255.255.255.0
 no shutdown

ip route 0.0.0.0 0.0.0.0 209.165.100.2

4.2 Licencia de seguridad habilitada

Para poder usar IPSec, fue necesario habilitar el paquete tecnológico SecurityK9. Después se guardó la configuración y se reinició el router para aplicar cambios. 

license boot module c1900 technology-package securityk9
copy running-config startup-config
reload
show version

5. Implementación de ACLs (Tráfico interesante)

Se configuró una ACL extendida para definir el tráfico que será cifrado por IPSec. En este caso, se permitió comunicación entre las redes privadas: 192.168.1.0/24 y 192.168.3.0/24. 

access-list 100 permit ip 192.168.1.0 0.0.0.255 192.168.3.0 0.0.0.255
access-list 100 permit ip 192.168.3.0 0.0.0.255 192.168.1.0 0.0.0.255

6. Phase 1: ISAKMP Policy

En esta fase se estableció la política ISAKMP, encargada de negociar parámetros de seguridad antes de crear el túnel IPSec. Se usó AES 256, autenticación por clave precompartida y grupo DH 5. 

crypto isakmp policy 10
 encryption aes 256
 authentication pre-share
 group 5

crypto isakmp key secretkey address 209.165.200.1

7. Phase 2: IPSec Transform-Set

Se configuró el transform-set para definir cómo se cifrarán y validarán los paquetes. Se utilizó ESP con AES 256 y SHA-HMAC para integridad. 

crypto ipsec transform-set R1->R2 esp-aes 256 esp-sha-hmac
crypto ipsec transform-set R2->R1 esp-aes 256 esp-sha-hmac

8. Crear el mapa criptográfico

El crypto map funciona como contenedor que une peer remoto, transform-set, PFS y la ACL. Se estableció un tiempo de vida de 24 horas (86400 segundos). 

crypto map IPSEC-MAP 10 ipsec-isakmp
 set peer 209.165.200.1
 set pfs group5
 set security-association lifetime seconds 86400
 set transform-set R1->R2
 match address 100

9. Aplicar el mapa criptográfico

Finalmente, el mapa criptográfico se aplicó en la interfaz conectada hacia la red pública, habilitando IPSec en esa interfaz. De esta forma, todo tráfico que coincida con la ACL será cifrado automáticamente. 

interface g0/0
 crypto map IPSEC-MAP

10. Resultado final

Una vez completada la configuración, se logró establecer el túnel IPSec entre ambas redes privadas, permitiendo tráfico seguro y cifrado a través del ISP.

Figura 2. Prueba de conectividad y funcionamiento del túnel IPSec.

11. Conclusión

En conclusión, implementar una VPN IPSec Tunnel requiere una configuración organizada y coherente. Primero se estableció conectividad básica mediante direccionamiento IP y rutas estáticas. Luego se habilitó la licencia SecurityK9 para activar funciones criptográficas. Posteriormente, se definió el tráfico interesante con ACLs y se configuraron las fases ISAKMP e IPSec. Finalmente, al aplicar el crypto map en la interfaz externa, se logró establecer una conexión segura, garantizando autenticación, integridad y confidencialidad entre las redes.
